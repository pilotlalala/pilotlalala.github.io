架构发展史

第一阶段 单机阶段

​ 浏览器 ——–>服务器tomcat

第二阶段 数据分离

​ 浏览器——>服务器——>数据库mysql

第三阶段 缓存应用

​ 浏览器—->服务器——>mysql

​ |——–>redis 缓存服务器

第四阶段 集群应用

​ 浏览器—->nginx 服务器 mysql mysql mysql mysql

​ —–> 服务器 —-> edis 缓存服务器

​ 服务器

nginx反向代理的软件 访问不同地址 可以反向访问到任意服务器 再去访问其他的东西

第五阶段 数据分离

​ 1.数据库优化

​ 读写分离：读取和写入数据库操作分离开

​ 2.分表分库

​ 浏览器—->nginx 服务器 mysql mysql mysql mysql

​ —–> 服务器 —-> edis 缓存服务器 edis 缓存服务器 edis 缓存服务器

​ 服务器

第六阶段 动静分离

​ 浏览器—->nginx 服务器 mysql

​ —–> 服务器 —-> edis 缓存服务器 edis 缓存服务器 edis 缓存服务器

​ 服务器

​

​ ——>静态资源 图片 css等 代表：七牛云 存储数据

第七阶段 跨机房代理（SSM技术 Spring（装配bean的工厂 调用构造方法来实例化对象）+SpringMVC（核心是servlet 拦截客户请求）+MyBatis(封装各种jdbc操作)）

浏览器—–>其他地区的机房——>lvs虚拟服务器—->nginx 服务器 mysql mysql mysql

​ —–> 服务器 —-> edis 缓存服务器 edis 缓存服 务器 edis 缓存服务器

​ 服务器

​

​ ——>静态资源 图片 css等 代表：七牛云 存储数据

第八阶段：技术革新（数据异构）

​ 以前是靠加服务器弥补技术短板

​ 因为加服务器的成本高于研发 所以得技术革新

​ 主要革新后面的存储技术 百花齐放

​ 浏览器—–>lvs+nginx集群+tomcat集群（业务集群）—–>查询殷勤 分布式文件集群 redis集群 mysql集群 各种Nosql等

​

第九阶段 业务线拆分

​ 浏览器—–>lvs+nginx集群——>B2B C2C B2C O2O等集群—–>查询引擎 分布式文件集群 redis集群 mysql集群 各 种Nosql等

第十阶段 公共服务拆分

​ 集群方面优化 复用 公共服务集群 短信业务 身份认证等等

​

第十一阶段 微服务：中心架构（进一步优化）

​ 1.业务架构师

​ 2.平台架构师

​ 3.数据架构师 早期DBA（数据库管理员）更适合

​ 浏览器—–>lvs+nginx集群——->产品中心——>服务总线——->业务：交易模块 人员米快 商品模块 公共：短信、认证 平台支撑：日志 自动测试—–>数据处理层：查询引擎 分布式文件集群 redis集群 mysql集群 各

第十二阶段 容器化时代

​ 服务器改成资源容器（容器化） 各种服务变成一个个资源点 然后需要运维K8s

大概是目标之类的东西
  • 用html+css+js、jq+servlet+jdbc+mysql做一个CRUD 要求两个小时以内
  • SSM整合+CRUD实现 要求2个小时以内完成
  • Excel，word，txt,xml,jsp数据库，一个JAVAbean互相传输数据 4个小时以内 （可以用工具 必须明白为什么）

  • 搭建一个快速研发的单机项目 SSM
    • 架构师
  • 秒杀 JVM优化 手写RPC spring源码全家桶 dubbo复写 netty复写等等
  • 简单的微服务架构平台搭建
