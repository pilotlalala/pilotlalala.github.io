JSP

jsp

jsp原理

jsp本质就是servlet 是使用java开发的web项目不可缺少的

jsp作用

jsp执行过程

客户端发送请求
jsp容器将jsp转成servlet源代码
进入servlet的执行过程
将结果响应给客户端

jsp原理

在tomcat的work文件夹
里面生成.java脚本文件并且编译成class文件
内容用out.write()写出来

jsp组成元素

注释元素

注释
 可见注释：<!-- -->
 	可以在查看源代码的时候看得见
 	ctrl+/
 程序员专用注释：<%-- --%>
 	查看源代码看不见
 单行注释：// 会以java代码显示在脚本文件中
 多行注释
 	/**
 	*
    **/

指令元素

<%page%>指令
	import="引入的包"
	
	errorPage 如果当前页面出现错误 跳转到对应页面 
	isErrorPage 在要跳转的那个页面中使用 默认值false 只有当设置成true的时候才能使用
															       exception.getMessage()
	exception.getMessage()方法获取详细的报错信息
	

<%include%>指令
 找不到的话可以<%@之后alt+/来找
 file:静态包含页面 表示要包含的页面 编译包含在被包含的页面中 不会生成其他文件
 
 
 
 <%taglib url="http://jaba.sun.com/jsp/jstl/cCore" prefix="c"%>指令
 	一般用来导入标签库 用于引入jsp的指令

脚本元素

代码段：<%%>
表达式：<%=%>获取值和方法
声明：<%！声明变量或者方法 %>  声明的是全局变量

动作元素

必须用jsp：这个前缀来标记

静态包含：上面的include指令 编译的时候就执行了
动态包含 <jsp:include page="">
		 <jsp:param value="" name="">可以带参数 在要跳转的页面上面直接request.getParameter("")																					就可以
        </jsp:include>
		包含的页面也会被编译 每次请求编译一次
		注意跳转后面不能加注释
		

forward跳转
 跳转到指定页面 后面的代码不会再执行
 把include改成forward
 
 
 
<jsp:userBean>创建bean实例

jsp九大内置对象

request  请求 接受HTTP协议
response 响应
session
application 知道服务器关闭 不然将信息保存在服务器中
config
pageContext
   pageContext对象的创建和初始化都是由容器来完成的
out
exception

 img.jsp里面写好
 然后在img标签的src引用
 然后js的点击事件里面 把路径改成 src="img.jsp?" + new Date()
