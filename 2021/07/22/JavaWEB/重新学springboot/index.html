重新学习Springboot

引入

刚开始实习 写了一个springboot项目

看到控制层的时候 被问到了关于PO VO DTO

完全不懂什么意思

赶紧解决问题然后学起来

关于PO VO BO DTO DAO POJO

先看定义再一点一点理解吧

PO

PO[persistant object]持久层对象，数据库中一条数据可以理解成一个持久层对象，持久层对象的属性和数据库中的字段是一一对应的

​ 因为ORM框架的广泛使用而被引入到JavaEE的项目中

​ 问题：持久层？ORM？

持久层

​ 持久[Persistence],即把数据（如内存中的对象）保存到可永久保存的存储设备中（如磁盘）。持久化的主要应用是将内存中的数据存储在关系型的数 据库中，当然也可以存储在磁盘文件中、XML数据文件中等等。

​ 持久层[Persistence Layer]，即专注于实现数据持久化应用领域的某个特定系统的一个逻辑层面，将数据使用者和数据实体相关联。

​ J2EE的三层结构指的就是表示层[Presentation]、业务逻辑层[Business Logic]、基础架构层[Infrastructure]

​ 开发者通常会对三层结构进行扩展来满足一些项目要求 一个最常用的扩展就是把三层变为五层

​ 表示层[Presentation] 控制、中间层[Controller/Mediator]、领域层[Domain]、数据持久层[Data Persistence]、数据源层[Data Source]

​ 简单来说就是对象—关系型数据库的一个映射

​ Mybatis就是一个持久层框架

ORM

一种设计技术

对象关系映射[Object Relational Mapping]

简单来说,它的作用是在关系型数据库和对象之间作一个映射，这样，我们在具体的操作数据库的时候，就不需要再去和复杂的SQL语句打交道，只要像平时操作对象一样操作它就可以了 。

所以PO应该就是一条数据库数据？

entity里面的映射表的对象是PO吗？查了一下好像不是的

似乎POJO才是entity

先看看POJO吧

POJO

POJO：[Plain Ordinary Java Object]，我理解为极其单纯的Java对象
一般只有属性字段，无参构造以及get和set方法，也是指那些没有从任何类继承、也没有实现任何接口，更没有被其它框架侵入的java对象。

POJO转PO 来自https://blog.csdn.net/qq_34532187/article/details/82858782

@GetMapping(value="/addgames")
   public void addGames() {
       Games game = new Games();//初始化一个POJO，或者entity
      //set值作为是持久化处理的过程
       game.setId(12);
       game.setName("龙珠超宇宙");
       game.setType("热血格斗");
       game.setSeries("七龙珠");
       game.setPrice(10.0);
       //调用saveOne方法进行持久化，保存进数据库，变为PO，代表一条表数据
       gamesMapper.saveOne(game);
   }
总结PO和POJO

所以POJO是一个空对象，没有任何数据，往里面装了数据就会变成PO，所以entity是PO好像也没错

VO

VO[Value Object/View Object]值对象。

如果称呼他为值对象（Value Object）的话，那门他可以理解为存放业务对象（BO）的一个地方。（？）

值对象也可以称作页面对象（View Object），如果称作页面对象，那么它所代表的就是整个页面表示层的对象

…

完 全 没 懂

继续看

PO 实体类对象：对应数据库的字段
DTO是数据传递对象：控制器接收前端数据模型，然后传递给servise
BO 业务对象：适合多查询的，比如多个PO组合起来的，或者多表查询返回的数据模型在servise里在处理成VO返回给前端
VO 是servise 返回的结果，给前端用的

https://www.jianshu.com/p/ff40c86a6f00

总结VO

也就是说 VO是用来封装需要对页面展示的值的对象，DTO是从前端接收到的值传给service来处理

DTO

DTO[Data Transfer Object]

百度百科：数据传输对象（DTO)(Data Transfer Object)，是一种设计模式之间传输数据的软件应用系统。数据传输目标往往是数据访问对象从数据库中检索数据。数据传输对象与数据交互对象或数据访问对象之间的差异是一个以不具有任何行为除了存储和检索的数据（访问和存取器）。

比如我们一张表有100个字段，那么对应的PO就有100个属性。

但是我们界面上只要显示10个字段，

客户端用WEB service来获取数据，没有必要把整个PO对象传递到客户端

问题：可是这样子不就和VO重复了吗
VO和DTO

VO和DTO

来自https://www.cnblogs.com/vegetableDD/p/11732495.html

1、entity 里的每一个字段，与数据库相对应，

2、vo 里的每一个字段，是和你前台 html 页面相对应，

3、dto 这是用来转换从 entity 到 vo，或者从 vo 到 entity 的中间的东西 。（DTO中拥有的字段应该是entity中或者是vo中的一个子集）

4、dto 和 entity 里面的字段应该是一样的，dto 只是 entity 到 vo，或者 vo 到 entity 的中间过程

总归vo是从前端拿到的字段组成的对象

dto是传输的时候用的对象 是的VO到POJO进而存到数据库中 这应该没错

至于具体怎么使用，再问一下吧

BO

BO[Business Object]

业务对象，一般用于业务层，当业务比较复杂的时候，多使用此对象封装多个对象进行传递

适合多查询的，比如多个PO组合起来的，或者多表查询返回的数据模型在servise里在处理成VO返回给前端

补充：关于类自动创建相关操作方法的注释

先引入lombok

<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.4</version>
    <scope>provided</scope>
</dependency>

然后就可以使用了

@Data注释可以自动创建get/set/g

@AllArgsConstructor ： 注在类上，提供类的全参构造
@NoArgsConstructor ： 注在类上，提供类的无参构造
@Setter ： 注在属性上，提供 set 方法
@Getter ： 注在属性上，提供 get 方法
@EqualsAndHashCode ： 注在类上，提供对应的 equals 和 hashCode 方法
@Log4j/@Slf4j ： 注在类上，提供对应的 Logger 对象，变量名为 log
