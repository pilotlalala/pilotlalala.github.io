版本控制

什么是版本控制 版本迭代

版本控制(Revision control)是在开发过程中，管理我们对于文件、目录、工程的修改历史，方便查看更改记录，并且备份以便于恢复以前的版本的工具

  • 多人协同工作
  • 追踪和记载一个或者多个文件的历史记录
  • 组织和保护你的源代码和文档
  • 统计工作量
  • 并行开发 提高开发效率

常见版本控制工具

常见版本控制工具

分类

  • 本地版本控制

    • 适合个人使用

  • 集中版本控制

    • 所有版本数据都存到服务器上 服务器一坏所有东西就都得丢 而且得联网才能看到历史版本 总归很麻烦
    • 得定期备份服务器 代表产品：SVN

  • 分布式版本控制Git

    • 所有版本信息仓库同步到所有用户本地上 只需要在有网络的情况下push到相应的服务器或者其他用户
    • 有风险！每个人拥有全部代码！
    • 不会因为网络或者服务器问题而造成工作不能的情况

以上两点就是SVN和Git的一个区别

可以说Git目前是最好的分布式版本控制系统

历史

Linux内核开源项目有着为数中广的参与者。绝大多数的维护工作都花在了提交补丁和版保存归档的工作上(1991-2001)。到了2002年，整个项目开始启用一个叫做bitkeeper的分布式管理系统来维护代码。

这个时候Linux开源社区和BitKeeper还是合作关系，毕竟BitKeeper要收费的，相当于是Bitkeeper在赞助Linux。但是Linux的社区中有很多很多大佬，在试图去破解BitKeeper核心代码，于是乎

2005年，开发BitKeeper的商业公司和Linux的开源社区的合作关系结束。他们收回了Linux开源社区免费使用BitKeeper的权力。

这就迫使Linux开源社区（尤其是Linux的创始人Linus Torvalds李纳斯·托瓦兹）基于使用BitKeeper的经验教训，开发出了自己的版本系统，也就是Git。

所以说Linus Torvalds是Linux和Git之父（1969 芬兰），很牛逼的是，这货开发出这玩意只用了两周左右。

Git是免费的、开源的。最初的目的是为了辅助Linux开发，代替BitKeeper。

Git配置

查看配置的命令：git config -l

查看系统自动配置的命令：git config --system --list

查看自己配置的命令：git config --global --list


Git的系统级配置文件gitconfig：Git/etc/gitconfig

Git的用户级配置文件：c:user/Administrator/.gitConfig

必要配置：配置用户名和邮箱

git config --global user.name "用户名"

git config --global user.email "你的邮箱"


环境变量可以不用配置，因为配置环境变量是为了在任何地方使用命令（就好像java命令一样）

但是Git自动配置了右键使用Git Gui和Git Bash
